# Proyecto-Fundamentos-de-base-de-datos
## Descripción del proyecto

El proyecto es un **sistema de gestión de un taller mecánico** que controla la entrada y salida de autos, las reparaciones realizadas y los cobros asociados.  
La base de datos registra a los clientes, sus vehículos, las órdenes de trabajo, los servicios/reparaciones aplicados, los repuestos usados y los pagos realizados por cada orden. 
Cada vehículo que entra al taller genera un registro con fecha y hora de entrada, diagnóstico, trabajos a realizar y responsable; cuando el trabajo termina, se registra la salida del auto, el detalle de lo cobrado y el estado de pago.  
El sistema permite consultar el historial de reparaciones por cliente o por vehículo y genera información clara para saber qué se hizo, cuánto costó y quién lo autorizó.

## Motivación del proyecto

La principal motivación es **evitar pérdidas y fraudes** en el manejo del dinero del taller.  
Al tener todos los autos, trabajos y cobros registrados en la base de datos, se reduce la posibilidad de que alguien cobre de más, cobre por fuera o no registre un servicio.  
El sistema facilita llevar una **contabilidad de autos y cobros por semana, mes y año**, generando reportes de ingresos, número de vehículos atendidos y servicios más frecuentes. 
Con esta información el dueño del taller puede detectar diferencias entre lo trabajado y lo cobrado, tomar mejores decisiones de negocio y aumentar la confianza de los clientes al tener todo documentado.
